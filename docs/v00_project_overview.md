# v00 프로젝트 개요

## 1. 프로젝트 배경

SolarTrack Agent는 소형 태양광 패널을 대상으로 태양 추적, 발전량 검증, 출력 저하 진단을 한 흐름으로 묶는 시뮬레이터다. 초기 목표는 실제 하드웨어가 없어도 React 화면에서 상태 변화와 Agent 판단을 설명할 수 있게 만드는 것이다.

## 2. 왜 Agent 구조인가

기존 2축 태양추적기는 이미 연구와 상용 사례가 있다. 이 프로젝트의 차별점은 패널을 움직인 뒤 발전량이 실제로 개선됐는지 검증하고, 개선되지 않으면 원인을 진단해 사용자에게 조치를 제안하는 데 있다.

## 3. 대상 설비

대상은 건물 옥상에 설치된 소형 미니 태양광 설비다. 대규모 발전소보다 구조가 단순하고, ESP32와 소형 센서로 실험 가능한 범위에 가깝다.

## 4. 전체 시스템 흐름

```text
태양 위치 계산
-> 가상 센서값 생성
-> 위치 기반 기상 정보 수집
-> 방위각 정렬
-> 고도각 정렬
-> 발전량 계산
-> 고정식 대비 비교
-> 진단 및 조치 추천
```

## 5. 최종 시스템 구성

| 영역 | 역할 |
|---|---|
| React | 시뮬레이션 화면과 사용자 조작 |
| FastAPI | 상태 계산, 진단, 외부 API 연동 |
| LangGraph | 진단 흐름과 리포트 생성 |
| Vision | 구름, 오염, 음영 후보 판단 |
| ESP32 | 센서 수집과 모터 제어 확장 |

## 6. 단계별 버전 계획

v00은 문서와 폴더 구조를 만든다. v01부터 React 시뮬레이터를 시작하고, v02에서 계산 모델을 분리한다. 이후 순차제어, 진단, 비전, 백엔드, LangGraph, 하드웨어 순서로 확장한다.

## 7. 위치 기반 기상 정보 수집 Agent 역할

기상 정보 Agent는 설치 위치의 구름량, 강수, 온도, 풍속, 일사 조건을 수집한다. 이 정보는 발전량 계수와 추적 보류 판단에 참고하되, 모터 제어 명령으로 직접 연결하지 않는다. API 실패 시에는 가상 시나리오 값으로 대체하고 로그에 남긴다.

## 8. 제외할 범위

- 대규모 태양광 발전소용 상용 시스템 구현
- 220V AC 인버터 실험
- LLM 직접 모터 제어
- 초기 단계의 실제 비전모델 학습
- 초기 단계의 실제 ESP32 하드웨어 연결
