# v01 React 시뮬레이터 계획

## 1. 목표

React 단독으로 SolarTrack Agent의 핵심 흐름을 눈으로 확인할 수 있는 시뮬레이터를 만든다. 실제 백엔드나 하드웨어 없이 태양 위치, 패널 각도, 센서값, 발전량, Agent 판단을 표시한다.

## 2. 화면 구성

| 영역 | 내용 |
|---|---|
| 상단 | 프로젝트명, 현재 시나리오, 실행 상태 |
| 중앙 | 태양과 패널 시각화 |
| 우측 | 제어 패널과 시나리오 선택 |
| 하단 | 발전량 그래프, Agent 로그 |

## 3. 주요 컴포넌트

- `SolarScene`: 태양 위치와 패널 방향 표시
- `ControlPanel`: 시작, 정지, 초기화, 수동 각도 조정
- `SensorPanel`: 조도, 전압, 전류, 온도 표시
- `PowerChart`: 고정식과 추적식 발전량 비교
- `AgentPanel`: 현재 단계, 진단, 추천 조치 표시
- `VisionPanel`: 가상 비전 추론 결과 표시
- `WeatherPanel`: 기상 정보와 추적 제한 사유 표시
- `LogPanel`: 시간순 이벤트 표시

## 4. 상태값 설계

`SolarState`는 시간, 태양 방위각, 태양 고도각, 패널 방위각, 패널 고도각, 센서값, 발전량, 기상 상태, 시나리오, Agent 단계, 진단 결과를 포함한다.

## 5. 시뮬레이션 루프

```text
시간 증가
-> 태양 위치 계산
-> 기상 상태 갱신
-> 센서값 계산
-> 자동 추적 여부 판단
-> 발전량 계산
-> 진단 결과 갱신
-> 화면 렌더링
```

## 6. 시나리오 버튼

`normal`, `cloudy`, `shade`, `soiling`, `overheat`, `charging_issue`, `overload`를 기본 시나리오로 둔다.

## 7. 고정식 vs 추적식 비교

고정식 패널은 방위각 0도, 고도각 35도로 둔다. 추적식 패널은 Agent가 조정한 현재 각도를 사용한다.

## 8. 기상 정보 표시 영역

초기에는 실제 API 대신 가상 기상 상태를 사용한다. 표시 항목은 날씨, 구름량, 강수 여부, 온도, 풍속, 추적 제한 여부다.
