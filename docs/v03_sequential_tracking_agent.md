# v03 순차제어 Agent

## 1. 순차제어가 필요한 이유

하부 방위각과 상부 고도각을 동시에 움직이면 상태 판단이 복잡해지고 초보자가 흐름을 이해하기 어렵다. 먼저 방위각을 정렬하고, 그 다음 고도각을 조정한다.

## 2. 하부 방위각 우선 정렬

태양 방위각과 패널 방위각 차이가 허용 범위보다 크면 하부 원판을 조금씩 회전한다.

## 3. 상부 고도각 후정렬

방위각이 충분히 맞으면 태양 고도각과 패널 고도각 차이를 기준으로 상부 패널을 조정한다.

## 4. 동시에 모터를 움직이지 않는 이유

동시 제어는 전력 소모와 진단 복잡도를 높인다. 순차제어는 원인과 결과를 분리해서 설명하기 쉽다.

## 5. 구름 감지 시 모터 동작 보류

구름으로 일시적으로 발전량이 떨어진 경우 각도 문제로 오판하지 않는다. 구름 상태에서는 불필요한 서보모터 동작을 보류한다.

## 6. 기상 정보 기반 보류 또는 감속

강수나 강풍이 감지되면 추적을 보류하거나 제한한다. 이 판단은 안전 규칙으로 처리하고, LLM이 직접 결정하지 않는다.

## 7. 각도 제한

방위각은 -90도에서 +90도, 고도각은 0도에서 70도 범위로 제한한다.

## 8. 제어 흐름

```text
센서값 확인
-> 기상 정보 확인
-> 구름/강수/강풍이면 hold
-> 안전하면 방위각 정렬
-> 방위각 정렬 완료 후 고도각 정렬
-> 발전량 검증
```

## 9. 의사코드

```text
if weather.rain or weather.highWind:
  phase = "hold"
else if cloudDetected:
  phase = "hold"
else if abs(sunAzimuth - panelAzimuth) > 3:
  adjust azimuth
else if abs(sunElevation - panelElevation) > 2:
  adjust elevation
else:
  verify power gain
```
