# v10 ESP32 하드웨어 계획

## 1. 확장 목표

React 시뮬레이터와 FastAPI 구조가 안정된 뒤 ESP32 기반 실제 센서와 모터 제어로 확장한다.

## 2. 센서 구성

LDR 4개 또는 BH1750으로 조도 방향을 측정한다. INA219로 전압/전류를 측정하고, DS18B20으로 패널 온도를 측정한다.

## 3. 하부 회전 구동부

하부 방위각은 스텝모터와 드라이버를 사용한다. 각도 범위와 속도 제한을 둔다.

## 4. 상부 패널 각도 구동부

상부 고도각은 고토크 서보모터를 사용한다. 0도에서 70도 범위를 넘지 않게 제한한다.

## 5. 카메라 모듈

구름, 오염, 음영 후보를 확인하기 위한 카메라 모듈을 고려한다.

## 6. 전원 구성

초기 실험은 저전압 DC 기반으로 제한한다. 220V AC 인버터 실험은 제외한다.

## 7. 통신 방식

ESP32는 Wi-Fi로 FastAPI 서버에 센서값을 전송하고, 서버는 검증된 제어 명령만 반환한다.

## 8. 안전 제한

강풍, 과열, 각도 초과, 센서 이상 상태에서는 모터 동작을 제한한다. LLM은 직접 제어하지 않는다.

## 9. v10 구현 상태

현재 단계에서는 실제 모터 제어를 실행하지 않고, 백엔드 안전 검증 경계와 ESP32 통신 예제만 구현했다.

| 항목 | 구현 내용 |
|---|---|
| 백엔드 프로필 | `GET /api/hardware/profile` |
| 텔레메트리 입력 | `POST /api/hardware/telemetry` |
| 안전 검증 | 비, 강풍, 과열, 저전압, 센서 이상, emergency stop |
| 명령 종류 | `hold`, `move` |
| 예제 펌웨어 | `hardware/esp32_solartrack_gateway/esp32_solartrack_gateway.ino` |
| 직접 제어 | 아직 수행하지 않음 |

ESP32는 센서값과 현재/목표 각도를 서버로 보내고, 서버가 제한값을 검증한 뒤 안전한 명령만 반환한다. 실제 핀 제어와 모터 드라이버 코드는 bench test 이후에 추가한다.
